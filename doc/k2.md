
## Kube-apiserver – Approfondimento pratico
 
 È il cuore del control plane.

Responsabilità:
  - Autenticazione
  - Autorizzazione
  - Validation requests
  - Communication hub cluster

### Architettura

Quando esegui:
```
kubectl get pods
```
Flusso:
```
kubectl client
↓
kube-apiserver
↓
etcd database
↓
scheduler / controller manager
```
- Verifica stato API server 
Controllare processo container
```
crictl ps | grep kube-apiserver
```
Se non c’è → restart control plane component.

Controllare logs
```
crictl logs <container-id>
```
Cerca:
  - certificate
  - port binding
  - file path
  - auth failure

Sintomi di kube-apiserver rotto

Se vedi:
```
Unable to connect to server
connection refused
TLS certificate error
CrashLoopBackOff
```

- Server endpoint corretto

Deve essere:
```
https://localhost:6443
```
oppure
```
https://controlplane:6443
```
- Certificati validi

Errore tipico:

certificate is valid for X, not Y

Significa:

- hostname mismatch

- Pod control plane healthy

Verifica namespace kube-system:
```
kubectl get pods -n kube-system
```
Se kubectl non funziona:

Prova:
```
export KUBECONFIG=/root/.kube/config
```
Oppure controlla context:
```
kubectl config current-context
```

| Comando                                  | Cosa fa                      | Quando usarlo               | Significato pratico                           |                                      |
| ---------------------------------------- | ---------------------------- | --------------------------- | --------------------------------------------- | ------------------------------------ |
| `crictl ps \| grep kube-apiserver`     | Mostra container API server  | Se API server sembra morto  | Se non appare → control plane broken |
| `crictl logs <container-id>`             | Legge logs runtime container | CrashLoopBackOff            | Prima cosa da fare nei lab                    |                                      |
| `kubectl get pods -n kube-system`        | Verifica control plane pods  | API server non risponde     | Scheduler/controller/coredns dipendono da lui |                                      |
| `systemctl status kubelet`               | Verifica node agent          | Cluster non operativo       | Se kubelet è down → pods non partono          |                                      |
| `openssl x509 -in cert.crt -text -noout` | Ispeziona certificato        | TLS error                   | Verifica SAN hostname                         |                                      |
| `kubectl config view`                    | Mostra kubeconfig            | Auth problemi               | Controlla server endpoint                     |                                      |
| `kubectl config current-context`         | Mostra context attivo        | Comandi kubectl falliscono  | Molto frequente negli esercizi                |                                      |
| `export KUBECONFIG=/root/.kube/config`   | Forza kubeconfig path        | Shell lab environment       | Risolve 30% errori iniziali                   |                                      |

Se kube-apiserver è ok devi poter eseguire: 
```
kubectl get nodes
kubectl get pods -A
kubectl describe pod -n kube-system kube-apiserver-*
```
