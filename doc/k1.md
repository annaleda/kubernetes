
### Kubeconfig – Approfondimento pratico

Il file kubeconfig serve per:

- Definire cluster Kubernetes
- Definire utenti e certificati
- Definire contesti di lavoro
- Stabilire il server API endpoint

File tipico:

```
~/.kube/config
```
Struttura logica

 - Un kubeconfig contiene 3 sezioni principali:

1. Cluster

Definisce dove connettersi.

Contiene:

 - API Server URL
 - Certificate Authority

Esempio:
 ```
clusters:
- name: kubernetes
  cluster:
    server: https://127.0.0.1:6443
    certificate-authority-data: xxxx
```
2. User (Credentials)

Può usare:

 - Client certificate authentication
 - Token authentication
 - Username/password (meno comune)

Esempio:

```
users:
- name: kubernetes-admin
  user:
    client-certificate: /root/admin.crt
    client-key: /root/admin.key
```
3. Context

Il context collega:

- Cluster
- User
- Namespace (opzionale)

Esempio:
```
contexts:
- name: developer
  context:
    cluster: kubernetes
    user: martin
```
| Comando                                                                                         | Scopo                                                 | Quando usarlo negli esercizi                             | Cosa controllare se fallisce                          |
| ----------------------------------------------------------------------------------------------- | ----------------------------------------------------- | -------------------------------------------------------- | ----------------------------------------------------- |
| `kubectl config view`                                                                           | Mostra l’intero kubeconfig (cluster, users, contexts) | Quando la challenge parla di kubeconfig o autenticazione | Path file kubeconfig, certificati, endpoint API       |
| `kubectl config use-context developer`                                                          | Cambia il context attivo                              | Quando l’esercizio richiede un user specifico            | Verificare che context esista                         |
| `kubectl config current-context`                                                                | Mostra context attualmente attivo                     | Debug rapido prima di eseguire comandi cluster           | Se il context è sbagliato → molti esercizi falliscono |
| `kubectl config get-contexts`                                                                   | Lista tutti i context disponibili                     | Quando devi verificare setup multi-user                  | Controllare cluster/user associati                    |
| `kubectl config set-context developer --cluster=kubernetes --user=martin`                       | Crea nuovo context                                    | Quando challenge richiede user specifico                 | Cluster name deve combaciare con kubeconfig           |
| `kubectl config set-credentials martin --client-certificate=martin.crt --client-key=martin.key` | Crea utente con certificate authentication            | Quando task parla di user authentication                 | File cert/key devono esistere                         |
| `kubectl config set-cluster kubernetes --server=https://127.0.0.1:6443`                         | Configura cluster endpoint                            | Quando API server port è specificata                     | Port 6443 è quasi standard                            |
| `kubectl config unset users.<user>.client-key`                                                  | Rimuove chiave configurazione                         | Quando ci sono errori certificato path                   | Utile per reset troubleshooting                       |
| `kubectl config set-context --current --namespace=development`                                  | Imposta namespace di default del context              | Molto utile nei lab CKAD                                 | Riduce necessità di scrivere `-n namespace`           |
| `kubectl config delete-context developer`                                                       | Cancella context                                      | Quando devi rifare configurazione                        | Verifica prima con `get-contexts`                     |
| `export KUBECONFIG=/root/.kube/config`                                                          | Forza uso kubeconfig                                  | Quando kubectl non trova config                          | Utile in shell lab environment                        |

| Test                              | Comando                                  |
| --------------------------------- | ---------------------------------------- |
| Verificare connessione API server | `kubectl get nodes`                      |
| Verificare certificati            | `openssl x509 -in cert.crt -text -noout` |
| Test kubeconfig path              | `echo $KUBECONFIG`                       |
| Controllare context attivo        | `kubectl config current-context`         |
